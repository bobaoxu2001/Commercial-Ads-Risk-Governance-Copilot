# Portfolio Summary

AdShield AI is a bilingual commercial ads risk governance copilot that converts real FTC fraud aggregates, CFPB consumer complaints, optional Meta Ad Library creatives, and summarized public policies into an explainable case-review and metric-diagnosis workflow. It demonstrates the strategy/operations bridge: evidence extraction, category design, confidence-aware routing, human feedback, and analytics that quantify coverage without pretending public priors are internal enforcement labels.

## LinkedIn project description

Built AdShield AI, a full-stack risk operations copilot for commercial ads governance. The system ingests real FTC and CFPB public data, optionally enriches with the official Meta Ad Library API, extracts bilingual risk evidence, retrieves summarized policy rules, routes cases through a human-review workflow, and diagnoses risk/category/coverage metrics in a recruiter-ready command center. The deterministic path works without paid APIs and the documentation is explicit about provenance, privacy, and what public data cannot prove.

## Interview answer: “Tell me about a project where you used AI to improve risk operations.”

I built AdShield AI to show how AI can improve risk operations without hiding judgment behind a model. I started with real public FTC fraud aggregates and CFPB complaints, then built a bilingual taxonomy for financial scams, misleading claims, health risks, gambling, advertiser integrity, and Mandarin evasion patterns. The system extracts evidence, retrieves the most relevant summarized policy rule, produces a structured risk recommendation, and sends ambiguous cases to a human queue. I also built diagnostic views for category mix, feature lift, spikes, escalation coverage, and estimated review time. The important design choice was separating public risk priors from enforcement truth: the tool supports triage, but it does not claim CFPB complaints are violating ads or invent precision metrics before reviewers provide labels.
